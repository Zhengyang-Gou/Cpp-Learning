# 19 - C++ 常量 (Const)

## 1. 核心概念 (Core Concepts)

在 C++ 中，`const` 某种意义上是一个“伪概念”。它在最终编译生成的机器码层面其实什么都不做。它完全是提供给**编译器**和**程序员**看的机制，用于在编译阶段强制执行规则，防止数据被意外修改，从而让代码更安全、更易读。

## 2. 常量与指针 (Const & Pointers)

当 `const` 关键字与指针结合使用时，根据 `const` 所在的位置不同（在星号 `*` 的左边还是右边），它限制的内容也会发生变化。

| 语法结构 | 概念 | 规则说明 | 代码示例 |
| :--- | :--- | :--- | :--- |
| `const int* a`<br>(或 `int const* a`) | **指向常量的指针** | **数据**本身不能改，但**指针的指向**可以改。 | `a = &other_var;` (合法)<br>`*a = 2;` (报错) |
| `int* const b` | **常量指针** | **指针的指向**不能改，但指向的**数据**可以被修改。 | `*b = 2;` (合法)<br>`b = &other_var;` (报错) |
| `const int* const c` | **指向常量的常量指针**| 数据和指针的指向**都不能改**。 | 全都报错 |

## 3. 类方法中的 Const (Const in Class Methods)

在编写面向对象的类时，`const` 关键字有着极其重要的作用。它可以被放在类成员方法的参数列表之后。

| 方法类型 | 约束规则 | 作用与意义 |
| :--- | :--- | :--- |
| **只读方法 (Getter)** | 必须加 `const` | 向编译器保证：调用这个方法**绝对不会修改**类的任何成员变量。如果尝试在方法内修改，编译器会直接报错。 |
| **修改方法 (Setter)** | 绝对不能加 `const` | 因为这些方法的本意就是去修改对象的状态（成员变量）。 |

> **最佳实践**：只要一个类方法不需要修改成员变量，就**一定要**把它标记为 `const`。这在将对象作为常量引用（`const Entity& e`）传递时尤为重要，否则你将无法调用它的任何方法。

## 示例代码

```cpp
/*
const 某种意义上是一个伪概念
它在最终编译生成的机器码中其实什么都不做
它完全是给编译器和写代码的人看的
*/
#include <iostream>

int main() {
    // 1. 声明一个不能改变的变量
    const int MAX_AGE = 90;

    // 2. 指向常量的指针：数据本身不能改，但指针的指向可以改
    const int* a = new int(5);
    a = (int*)&MAX_AGE;

    // 3. 常量指针：指针的指向不能改，但数据本身可以改
    int* const b = new int(5);
    *b = 2;

    // 4. 指向常量的常量指针：都不能改
    const int* const c = new int(5);

}

class Entity {
// 在写类的时候，一定要把只读方法标记为 const
private:
    int m_X, m_Y;
public:
    // 这是一个 getter，它只读取数据，不修改数据，所以加上 const
    int GetX() const {
        // 如果你在这里尝试修改，编译器会直接报错
        return m_X;
    }

    // 这是一个 setter，它需要修改数据，所以绝对不能加 const
    void SetX(int x) {
        m_X = x;
    }
};
```
