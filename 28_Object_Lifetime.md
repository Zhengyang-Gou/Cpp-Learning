# 28 - C++ 对象生命周期 (Object Lifetime)

## 1. 核心概念 (Core Concepts)

在 C++ 中，对象的生命周期（即它在内存中存活的时间）主要取决于它被创建在哪里。C++ 的对象主要存在于两个地方：**栈 (Stack)** 和 **堆 (Heap)**。理解并控制对象的生命周期是 C++ 内存管理的核心。

## 2. 栈与堆的生命周期对比 (Stack vs. Heap Lifetime)

| 分配方式 | 代码形式 | 生命周期控制 | 销毁时机与机制 |
| :--- | :--- | :--- | :--- |
| **栈分配 (Stack)** | `Entity e;` | **自动管理** | 对象受限于其声明所在的**作用域 (Scope, 即 `{}` 内部)**。一旦程序的执行流离开了这个作用域，栈上的对象就会被**自动销毁**（自动调用析构函数）。 |
| **堆分配 (Heap)** | `new Entity();`| **手动管理** | 对象在堆上创建后会一直存活，完全无视作用域的限制。只有当程序员明确、手动地调用 `delete` 关键字时，它才会被销毁。如果忘记调用，就会导致**内存泄漏**。 |

> **最佳实践与基于作用域的资源管理 (RAII)**：
> 栈分配的自动销毁特性极其强大。C++ 程序员经常利用这一特性（即 RAII 机制）来管理底层的堆内存或其他资源（如智能指针 `std::unique_ptr` 内部封装了堆内存，但智能指针本身分配在栈上，当栈上的智能指针离开作用域死亡时，它的析构函数会自动 `delete` 内部的堆内存）。

---

## 示例代码

```cpp
// C++对象主要存在堆和栈两个地方
// 栈上的对象会在离开作用域时自动销毁，而堆上的对象需要手动管理内存
// 堆对象是手动管理生命周期的

#include <iostream>

class Entity
{
public:
    Entity() { std::cout << "Created\n"; }
    ~Entity() { std::cout << "Destroyed\n"; }
};

int main()
{
    // ==========================================
    // 1. 栈对象生命周期演示
    // ==========================================
    {
        Entity e; // 在栈上创建对象，打印 "Created"
        
    } // <-- 离开这个作用域时，e 自动死亡，打印 "Destroyed"

    // ==========================================
    // 2. 堆对象生命周期演示
    // ==========================================
    Entity* a = new Entity(); // 在堆上创建对象，打印 "Created"
    
    // 即使离开了创建它的函数或作用域，堆对象依然存活！
    // 必须手动调用 delete 来销毁它
    delete a; // 触发析构函数，打印 "Destroyed"
    
    return 0;
}
```
