# 17 - C++ 数组 (Arrays)

## 1. 内存分布 (Memory Layout)

在需要存储多个相同类型的数据时，普通的单独声明变量会导致数据在内存中散落分布。而数组会在内存（通常是栈内存）上分配一块**连续的内存区域**。

| 存储方式 | 内存状态 | 说明 |
| :--- | :--- | :--- |
| **独立变量** | 散落 | 像 `int a, b, c, d, e;` 这样声明的 5 个变量，在内存中的地址是不连续的。 |
| **数组** | 连续 | `int example[5];` 会在内存中分配一块刚好能容纳 5 个整数的连续空间。 |

## 2. 数组与指针的关系 (Arrays & Pointers)

在 C++ 中，数组名和指针有着极其紧密的底层联系，数组的索引操作本质上就是指针运算。

| 概念 | 机制说明 | 代码表现 |
| :--- | :--- | :--- |
| **首地址指针** | 数组名本质上是指向这块连续内存首地址（即第 0 个元素）的指针。 | `example` 等同于 `&example[0]` |
| **底层索引访问**| 使用 `[]` 访问数组元素时，编译器底层会将其等价翻译为指针的偏移与解引用。 | `example[2] = 5;` 相当于 `*(example + 2) = 5;` |

## 3. 数组长度计算与“退化”现象 (Array Size & Decay)

计算数组长度是一个常见需求，但必须注意数组作用域以及作为参数传递时的“退化”问题。

| 场景 | 特点 / 原理 | 代码示例 |
| :--- | :--- | :--- |
| **本地计算** | 在定义数组的同一个作用域内，可以用总字节数除以单元素的字节数来计算个数。 | `sizeof(example) / sizeof(int)` |
| **参数传递 (退化)**| 当数组作为参数传入函数时，它会**退化 (Decay)** 为普通指针（如 `int*`）。此时 `sizeof` 只能测出指针本身的大小（通常为 8 字节），无法再算出原数组长度。 | `void PrintArray(int* arr)` |

---

## 示例代码

```cpp
#include <iostream>

// 当你把数组传进函数时，它不再是数组了，它退化成了一个普通的 int* 指针
void PrintArray(int* arr) {
    // ⚠️ 警告：这里的 sizeof(arr) 测出的是一个指针的大小（64位系统下通常是 8 字节）
    // 8 / 4 = 2，根本得不到原数组正确的长度！
    // 正确的做法是：在传数组的同时，把数组长度作为第二个参数传进来。
    int count = sizeof(arr) / sizeof(int); 
}

int main()
{
    // 1. 创建包含 5 个整数的数组，在栈上分配连续内存
    int example[5]; 

    // 2. example 是指向这块连续内存首地址的指针，即指向 example[0]
    // 这种写法会被翻译成指针运算：*(example + 2) = 5
    example[2] = 5; 

    // 3. 正确计算本地作用域内数组元素的个数
    int count = sizeof(example) / sizeof(int);
    std::cout << "数组的元素个数: " << count << std::endl;

    std::cin.get();
    return 0;
}
```
