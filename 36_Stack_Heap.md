# 54 - C++ 栈内存 vs 堆内存 (Stack vs Heap Memory)

## 1. 基本概念 (Basic Concepts)

在 C++ 中，程序运行时使用的内存并不是一整块没有区别的空间。  
其中最常被提到的两部分就是：

1. **栈内存（Stack Memory）**
2. **堆内存（Heap Memory）**

这一集的核心是理解：

> **变量到底存在哪里？它们的生命周期由谁管理？为什么有时候要用 `new`，有时候根本不该用？**

---

## 2. 栈和堆的核心区别

### 栈内存（Stack）

栈内存通常用于：

- 函数调用
- 局部变量
- 函数参数
- 临时数据

它的特点是：

- **由编译器自动管理**
- 进入作用域时分配
- 离开作用域时自动释放
- 分配和释放速度非常快

---

### 堆内存（Heap）

堆内存通常用于：

- 动态分配对象
- 运行时才知道大小的数据
- 需要跨作用域存活的对象

它的特点是：

- **由程序员手动申请**
- 通常用 `new` 分配
- 通常用 `delete` 释放
- 生命周期不由作用域自动决定

---

## 3. 栈内存的工作方式 (How Stack Memory Works)

栈的使用方式很简单：

```cpp
int value = 5;
```

这里的 `value` 一般就是分配在栈上的。

当函数执行结束时，  
这个变量会随着函数栈帧（Stack Frame）一起被清理掉。

### 示例代码

```cpp
#include <iostream>

void Function()
{
    int a = 5;
    int b = 10;

    std::cout << a + b << std::endl;
}

int main()
{
    Function();
    return 0;
}
```

### 说明

在 `Function()` 执行时：

- `a`
- `b`

这些局部变量通常都在栈上。

当 `Function()` 返回后：

- 这一层函数调用结束
- 对应的栈帧被弹出
- 局部变量自动销毁

### 栈的关键特点

- 分配快
- 释放快
- 自动管理
- 生命周期短
- 非常适合局部数据

---

## 4. 栈帧 (Stack Frame)

每次调用函数时，程序通常会在栈上创建一块属于这个函数的区域，  
这块区域就叫做**栈帧**。

栈帧里通常会放：

- 函数参数
- 局部变量
- 一些调用信息

例如：

```cpp
void Multiply(int a, int b)
{
    int result = a * b;
}
```

调用 `Multiply(2, 3)` 时，栈上会为这个函数开一块空间，里面可能包含：

- 参数 `a`
- 参数 `b`
- 局部变量 `result`

函数结束后，这整块空间一起被清掉。

---

## 5. 堆内存的工作方式 (How Heap Memory Works)

如果我们不希望对象随着作用域结束而自动销毁，  
或者我们需要在运行时动态决定大小，  
就会使用堆内存。

### 示例代码

```cpp
#include <iostream>

int main()
{
    int* data = new int;
    *data = 5;

    std::cout << *data << std::endl;

    delete data;
    return 0;
}
```

### 说明

这里发生了几件事：

1. `new int` 在堆上分配了一块能存放 `int` 的内存
2. 返回这块内存的地址
3. 用指针 `data` 保存这个地址
4. 通过 `*data` 访问堆上的值
5. 最后用 `delete` 手动释放内存

### 重点

```cpp
int* data = new int;
```

这里：

- `data` 这个指针变量本身通常在**栈上**
- 但它指向的那个 `int` 在**堆上**

这是理解堆内存最重要的一点之一。

---

## 6. 栈和堆不是同一回事

很多初学者容易混淆：

```cpp
int* value = new int;
```

这行代码里有两个东西：

### 1）指针变量 `value`

它是一个普通局部变量，  
通常放在**栈上**。

### 2）`new int` 分配出来的对象

它是真正动态分配的那块内存，  
放在**堆上**。

所以：

> **“有指针”不等于“东西就在堆上”。**
>
> 关键要看对象本身是怎么分配出来的。

---

## 7. 堆内存的优点 (Advantages of Heap Memory)

### 1）生命周期更灵活

堆对象不会因为离开某个局部作用域就自动消失。

### 2）适合动态大小的数据

有些数据大小在编译时不知道，只能在运行时决定。

### 3）适合需要跨函数、跨作用域存在的对象

例如某些资源、共享数据结构、大对象等。

---

## 8. 堆内存的缺点 (Disadvantages of Heap Memory)

### 1）需要手动管理

如果用了 `new`，通常就要对应 `delete`。

### 2）容易造成内存泄漏 (Memory Leak)

如果忘了 `delete`，那块内存就一直占着，程序再也拿不回来。

### 3）分配速度通常比栈慢

堆分配要经过更复杂的内存管理过程，  
不像栈那样只是简单移动指针。

### 4）更容易出错

比如：

- 野指针
- 重复释放
- 访问已释放内存
- 内存泄漏

---

## 9. 内存泄漏示例 (Memory Leak Example)

### 错误示例

```cpp
#include <iostream>

int main()
{
    int* data = new int;
    *data = 10;

    std::cout << *data << std::endl;

    return 0;
}
```

### 问题

这段代码中：

- 分配了堆内存
- 但没有 `delete data;`

结果就是：

- 程序失去了对那块内存的正确管理
- 这块内存没有被显式释放
- 就形成了**内存泄漏**

### 正确写法

```cpp
#include <iostream>

int main()
{
    int* data = new int;
    *data = 10;

    std::cout << *data << std::endl;

    delete data;
    return 0;
}
```

---

## 10. 动态数组与堆内存 (Dynamic Arrays on the Heap)

堆内存常见用途之一就是动态数组。

### 示例代码

```cpp
#include <iostream>

int main()
{
    int size = 5;
    int* array = new int[size];

    for (int i = 0; i < size; i++)
        array[i] = i * 2;

    for (int i = 0; i < size; i++)
        std::cout << array[i] << std::endl;

    delete[] array;
    return 0;
}
```

### 说明

这里：

```cpp
int* array = new int[size];
```

表示在堆上分配一个长度为 `size` 的数组。

释放时必须使用：

```cpp
delete[] array;
```

而不是普通的：

```cpp
delete array;
```

---

## 11. 栈 vs 堆 对比表

| 特性 | 栈内存 (Stack) | 堆内存 (Heap) |
| :--- | :--- | :--- |
| **管理方式** | 编译器自动管理 | 程序员手动管理 |
| **分配速度** | 很快 | 通常较慢 |
| **释放方式** | 离开作用域自动释放 | 需要 `delete` / `delete[]` |
| **生命周期** | 通常较短，跟随作用域 | 更灵活，可跨作用域 |
| **容量** | 通常较小 | 通常更大 |
| **使用难度** | 简单、安全 | 更灵活，但更容易出错 |
| **常见用途** | 局部变量、函数参数 | 动态对象、动态数组、长期存在的数据 |

---

## 12. 为什么栈更快 (Why Stack Is Faster)

栈快的核心原因不是“它在物理上更高级”，  
而是它的管理方式更简单。

通常来说，栈分配只需要：

- 移动一下栈指针
- 或者在编译期就确定布局

而堆分配通常要考虑：

- 当前有哪些空闲内存块
- 多大空间可用
- 如何避免冲突
- 之后如何回收

所以：

> **栈快，是因为它简单；堆慢，是因为它灵活。**

---

## 13. 什么时候该用栈？什么时候该用堆？

### 优先用栈的情况

当对象：

- 生命周期很短
- 只在当前函数里使用
- 大小固定
- 不需要跨作用域保存

例如：

```cpp
int value = 10;
float speed = 3.5f;
char buffer[256];
```

这些通常都非常适合放在栈上。

---

### 考虑用堆的情况

当对象：

- 需要在运行时决定大小
- 需要跨函数、跨作用域存在
- 太大，不适合放在栈上
- 需要手动控制生命周期

例如：

- 动态数组
- 复杂数据结构
- 长时间存活的资源对象

---

## 14. 核心思想

### 重点 1：栈和堆的区别，首先是“管理方式”不同

- 栈：自动管理
- 堆：手动管理

---

### 重点 2：局部变量通常在栈上

函数里的普通局部变量，  
默认首先想到的是栈。

---

### 重点 3：`new` 分配的是堆内存

只要看到：

```cpp
new
```

就要意识到：

- 这是动态分配
- 需要考虑释放
- 可能有泄漏风险

---

### 重点 4：指针在栈上，不代表指向的对象也在栈上

这个区分非常重要。

---

### 重点 5：堆更灵活，但代价是复杂和风险

你得到的是更长的生命周期和更动态的能力，  
代价是更难管理、更容易犯错。

---

## 15. 实战建议 

### 1）能放栈上就先放栈上

因为：

- 简单
- 自动释放
- 不容易泄漏
- 性能通常更好

### 2）只有确实需要动态生命周期时再用堆

不要因为“看起来高级”就滥用 `new`。

### 3）只要用了 `new`，就必须立刻想到对应的释放

```cpp
new  -> delete
new[] -> delete[]
```

### 4）现代 C++ 更推荐用容器和智能指针管理堆资源

实际工程里，很多时候不会直接手写裸 `new` / `delete`，  
而会更多使用：

- `std::vector`
- `std::string`
- `std::unique_ptr`
- `std::shared_ptr`

但重点仍然是先把**栈和堆的底层概念理解清楚**。
